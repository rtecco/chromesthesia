## 3. Audio Synthesis Design (the heart of the project)

This is the most important section. The spec says **"interesting and textured sound is more important than absolute fidelity to the color metaphor."**

### 3a. Voice Architecture (color = timbre)

Each color in the palette maps to a **voice recipe** — not a single oscillator, but a small subtractive-additive hybrid patch:

| Palette Color | Base Pitch Region | Oscillator Stack | Filter Character |
|---|---|---|---|
| Cadmium Red | Low-mid (C3-G3) | 3 detuned saws + 1 sub square | Warm low-pass, high resonance — growling, brassy |
| Cobalt Blue | Mid (G3-D4) | 2 triangles + 1 sine, wide detune | Band-pass sweep — hollow, woody, clarinet-like |
| Lavender | High-mid (D4-A4) | 4 sines in harmonic series (1, 2, 3, 5) | Gentle high-shelf — glassy, bell-like, ethereal |
| Ochre/Gold | Low (G2-C3) | 2 squares + noise burst | Low-pass with envelope — percussive, earthy, tabla-like |

Each voice is built with `OscillatorNode` + `GainNode` per partial, routed through a `BiquadFilterNode`, then into the brush effects chain, then master.

**Detuning for richness:** Each oscillator in a stack is detuned by +/- 3 to 12 cents randomly, creating natural chorusing. This alone makes the sound far more interesting than a single oscillator.

**Noise layer:** Every voice mixes in a small amount of filtered noise (band-passed around the voice's fundamental), simulating the "breath" or "grit" of a real instrument. This is critical for texture.

### 3b. Position-to-Modulation Mapping

**X-axis (brightness):**
- X=0 (left): filter cutoff low, darker harmonics, lower detune spread
- X=1 (right): filter cutoff high, brighter harmonics, wider detune spread
- Implementation: `BiquadFilterNode.frequency` is mapped to `200 + x * 4000` Hz (exponential scale)
- Additionally, the amplitude of upper partials increases with X

**Y-axis (darkness/depth):**
- Y=0 (top): higher pitch register, less reverb, more presence
- Y=1 (bottom): lower pitch register, more reverb, more sub-bass
- Implementation: base pitch is shifted down by up to an octave as Y increases. Reverb wet/dry mix increases with Y.

**Stroke velocity** (derived from consecutive point timestamps + distance):
- Faster strokes = sharper attack envelope, more noise burst
- Slower strokes = gentler attack, more sustained tone

### 3c. Brush Type = Effects Chain

| Brush Type | Audio Effect | Implementation |
|---|---|---|
| Oil Flat | Rich reverb (cathedral-like) | `ConvolverNode` with a synthetically generated IR (no external file). Wet mix ~40% |
| Oil Round | Chorus + slight vibrato | Two detuned copies with LFO-modulated delay (~5ms, 2Hz) via `DelayNode` + `OscillatorNode` as LFO |
| Palette Knife | Distortion/saturation + short slap delay | `WaveShaperNode` with tanh curve + `DelayNode` (80ms, 30% feedback) — aggressive, edgy |
| Dry Brush | Granular stutter + bitcrusher-like | Rapid gain modulation (20-60Hz tremolo via `OscillatorNode` → `GainNode`) — gritty, broken, textural |

### 3d. Playhead / Sequencer

The loop playback works as a **vertical playhead scanning left to right** across the canvas:

1. The canvas is logically divided into N time-columns (e.g., 128 columns for a 4-second loop at ~32 columns/sec).
2. On play, a scheduling loop (using `AudioContext.currentTime` for precise timing, not `setInterval`) advances the playhead column by column.
3. For each column, the system queries which strokes pass through it by checking normalized X ranges of stroke points.
4. For each stroke segment intersecting the current column, a voice is triggered with parameters derived from the stroke's color, brush, Y position, and local direction.
5. Voices have short attack/release envelopes (50-150ms) so overlapping columns create smooth sustained tones for continuous strokes.
6. A visual playhead line is drawn on the canvas during playback.

**Loop length:** I recommend **4 seconds** as default. Short enough to feel like a rhythmic loop, long enough for musical phrasing. The spec says "TBD" — this can be made configurable later.

### 3e. Synthetic Impulse Response (no external dependencies)

Rather than importing a reverb IR file, we generate one programmatically:

```
function generateIR(context: AudioContext, duration: number, decay: number): AudioBuffer
```

Fill an `AudioBuffer` with exponentially decaying noise, optionally with early reflections modeled as delayed noise bursts. This keeps the project zero-dependency for audio.

## 4. Canvas Rendering — Oil Paint Texture

### Approach

Use **Canvas2D** (not WebGL — simpler, sufficient, fewer deps). Oil paint texture is achieved through:

1. **Multiple offset stamps per point:** For each recorded point, draw 3-5 slightly offset circles/ellipses with varying opacity (0.3-0.7). This simulates paint thickness variation.

2. **Bristle simulation:** For flat and round brushes, draw 8-15 thin parallel strokes (1-2px wide lines) that follow the stroke path but with slight random perpendicular offset. This creates visible bristle marks like the reference image.

3. **Color mixing at overlap:** Use `globalCompositeOperation = 'multiply'` or custom alpha blending when strokes overlap to simulate pigment mixing (not light mixing).

4. **Pressure → opacity + width:** Pointer pressure (from PointerEvent API, with fallback) controls both stroke width and opacity, simulating paint load on brush.

5. **Brush-specific texture patterns:**
   - **Oil Flat:** Wide, rectangular stamp with horizontal bristle lines
   - **Oil Round:** Circular stamp with radial bristle variation
   - **Palette Knife:** Sharp-edged, semi-transparent smear with directional stretching
   - **Dry Brush:** Sparse dots/dashes with large gaps — "skipping" effect

### Render Loop

The render loop is simple: on each frame, if the user is currently drawing, render the newest points of the active stroke. The full painting is kept as an offscreen canvas (`OffscreenCanvas` or a second `<canvas>` element) that is composited to the display canvas each frame. The playhead line is drawn on top during playback.

## 7. Key Design Decisions and Trade-offs

**Why no external audio library (Tone.js, etc.)?** The spec says minimize deps and ask before importing. The Web Audio API is sufficient for everything described here. Tone.js would add convenience but also 100KB+ and an abstraction layer that makes it harder to do the custom voice/effect routing we need. I recommend staying with raw Web Audio API.

**Why OffscreenCanvas for rendering?** Painting needs to be persistent (strokes accumulate), but we also need to overlay the playhead. Having the painting on an offscreen buffer that composites to the display canvas each frame lets us draw the playhead without re-rendering all strokes every frame.

**Why 4-second loops?** Short enough to be rhythmic and satisfying when looping, long enough to paint a phrase. This maps the canvas width to 4 seconds, giving approximately 32 time-slices per second at 128 columns — enough temporal resolution for smooth sound without excessive computation.

**Why normalize coordinates to 0..1?** Makes the JSON format resolution-independent. A painting created on a 1920px-wide screen plays back identically on a 768px screen. Also keeps the JSON compact.

**Why synthetic reverb IR instead of a file?** Zero external dependencies. A generated IR with exponentially decaying noise sounds perfectly adequate for this use case and avoids needing to host/load a .wav file.

**Why no `setInterval` for the playhead?** `setInterval` and `setTimeout` are jittery (can be 10-50ms late). The Web Audio API's clock (`AudioContext.currentTime`) is sample-accurate. The playhead scheduler uses a "look-ahead" pattern: a `setInterval` at ~25ms checks the current time and schedules audio events slightly in the future using `AudioContext.currentTime`, giving both accuracy and visual sync.

## 8. Potential Challenges

1. **Audio click/pop artifacts:** Voices must have attack and release envelopes (at least 10ms each) to avoid clicks. The `GainNode` ramp methods (`linearRampToValueAtTime`, `exponentialRampToValueAtTime`) handle this.

2. **Polyphony management:** A dense painting could trigger dozens of simultaneous voices when the playhead crosses it. Need a voice limit (e.g., 16 max) with oldest-voice stealing.

3. **iOS AudioContext restrictions:** Safari requires `AudioContext.resume()` inside a user gesture handler. The Play button handler must call this.

4. **Canvas performance with many strokes:** If the painting has thousands of points, re-rendering from scratch is expensive. The offscreen canvas approach (render each stroke once to the buffer) solves this. Only new points are incrementally drawn.

5. **Brush texture quality:** Achieving convincing oil paint texture with Canvas2D alone requires careful tuning of the bristle simulation. If the result is unsatisfying, we could consider a very small texture image (drawn programmatically once to a pattern canvas) tiled along the stroke path.

export const rgbToHsl = (r: number, g: number, b: number): [number, number, number] => {
  r /= 255; g /= 255; b /= 255;
  const max = Math.max(r, g, b), min = Math.min(r, g, b);
  const l = (max + min) / 2;
  if (max === min) return [0, 0, Math.round(l * 100)];
  const d = max - min;
  const s = l > 0.5 ? d / (2 - max - min) : d / (max + min);
  let h = 0;
  if (max === r) h = ((g - b) / d + (g < b ? 6 : 0)) / 6;
  else if (max === g) h = ((b - r) / d + 2) / 6;
  else h = ((r - g) / d + 4) / 6;
  return [Math.round(h * 360), Math.round(s * 100), Math.round(l * 100)];
};

import type { StrokePoint, Stroke, BrushType, PaletteColor } from '../types';

export type StrokeRecorderCallbacks = {
  onStrokeStart: (stroke: Stroke) => void;
  onStrokePoint: (strokeId: string, point: StrokePoint) => void;
  onStrokeEnd: (strokeId: string) => void;
  getActiveColor: () => PaletteColor;
  getActiveBrush: () => BrushType;
};

export function initStrokeRecorder(
  canvas: HTMLCanvasElement,
  callbacks: StrokeRecorderCallbacks,
): () => void {
  let activeStroke: { id: string; startTime: number; points: StrokePoint[] } | null = null;

  function normalizePoint(e: PointerEvent): StrokePoint {
    const rect = canvas.getBoundingClientRect();
    return {
      x: (e.clientX - rect.left) / rect.width,
      y: (e.clientY - rect.top) / rect.height,
      pressure: e.pressure > 0 ? e.pressure : 0.5,
      timestamp: activeStroke ? performance.now() - activeStroke.startTime : 0,
    };
  }

  function onPointerDown(e: PointerEvent) {
    if (e.button !== 0) return;
    canvas.setPointerCapture(e.pointerId);

    const id = crypto.randomUUID();
    const startTime = performance.now();
    const point = normalizePoint({ ...e, pressure: e.pressure } as PointerEvent);

    activeStroke = { id, startTime, points: [point] };

    const stroke: Stroke = {
      id,
      color: callbacks.getActiveColor(),
      brush: callbacks.getActiveBrush(),
      points: [point],
    };
    callbacks.onStrokeStart(stroke);
  }

  function onPointerMove(e: PointerEvent) {
    if (!activeStroke) return;
    const point = normalizePoint(e);
    activeStroke.points.push(point);
    callbacks.onStrokePoint(activeStroke.id, point);
  }

  function onPointerUp(_e: PointerEvent) {
    if (!activeStroke) return;
    callbacks.onStrokeEnd(activeStroke.id);
    activeStroke = null;
  }

  canvas.addEventListener('pointerdown', onPointerDown);
  canvas.addEventListener('pointermove', onPointerMove);
  canvas.addEventListener('pointerup', onPointerUp);
  canvas.addEventListener('pointercancel', onPointerUp);

  return () => {
    canvas.removeEventListener('pointerdown', onPointerDown);
    canvas.removeEventListener('pointermove', onPointerMove);
    canvas.removeEventListener('pointerup', onPointerUp);
    canvas.removeEventListener('pointercancel', onPointerUp);
  };
}

function rgbStr(c: PaletteColor, alpha: number): string {
  const [r, g, b] = c.rgb;
  return `rgba(${r},${g},${b},${alpha})`;
}

function strokeAngle(prev: StrokePoint, curr: StrokePoint): number {
  return Math.atan2(curr.y - prev.y, curr.x - prev.x);
}

function strokeSpeed(prev: StrokePoint, curr: StrokePoint): number {
  const dx = curr.x - prev.x;
  const dy = curr.y - prev.y;
  const dt = Math.max(curr.timestamp - prev.timestamp, 1);
  return Math.sqrt(dx * dx + dy * dy) / dt;
}

// Seeded-ish random from two floats, for deterministic bristle placement
function pseudoRand(a: number, b: number): number {
  const x = Math.sin(a * 12.9898 + b * 78.233) * 43758.5453;
  return x - Math.floor(x);
}

export function createRenderer(canvas: HTMLCanvasElement): CanvasRenderer {
  const ctx = canvas.getContext('2d')!;
  const bufferCanvas = document.createElement('canvas');
  const bufferCtx = bufferCanvas.getContext('2d')!;

  let playheadX: number | null = null;
  let animFrameId = 0;

  function syncSize() {
    const dpr = window.devicePixelRatio || 1;
    const rect = canvas.getBoundingClientRect();
    const w = Math.round(rect.width * dpr);
    const h = Math.round(rect.height * dpr);

    if (canvas.width !== w || canvas.height !== h) {
      // Save buffer contents before resize
      const tmpCanvas = document.createElement('canvas');
      tmpCanvas.width = bufferCanvas.width;
      tmpCanvas.height = bufferCanvas.height;
      const tmpCtx = tmpCanvas.getContext('2d')!;
      tmpCtx.drawImage(bufferCanvas, 0, 0);

      canvas.width = w;
      canvas.height = h;
      bufferCanvas.width = w;
      bufferCanvas.height = h;

      // Restore buffer contents scaled to new size
      if (tmpCanvas.width > 0 && tmpCanvas.height > 0) {
        bufferCtx.drawImage(tmpCanvas, 0, 0, w, h);
      }
    }
  }

  function compositeFrame() {
    ctx.clearRect(0, 0, canvas.width, canvas.height);
    ctx.drawImage(bufferCanvas, 0, 0);

    // Draw playhead
    if (playheadX !== null) {
      const px = playheadX * canvas.width;
      ctx.save();
      ctx.strokeStyle = 'rgba(0, 0, 0, 0.25)';
      ctx.lineWidth = 2;
      ctx.setLineDash([6, 4]);
      ctx.beginPath();
      ctx.moveTo(px, 0);
      ctx.lineTo(px, canvas.height);
      ctx.stroke();
      ctx.restore();
    }

    animFrameId = requestAnimationFrame(compositeFrame);
  }

  syncSize();
  compositeFrame();

  return {
    canvas,
    bufferCanvas,

    clear() {
      bufferCtx.clearRect(0, 0, bufferCanvas.width, bufferCanvas.height);
    },

    drawSegment(stroke: Stroke, prev: StrokePoint, curr: StrokePoint) {
      const rc: BrushRenderContext = {
        ctx: bufferCtx,
        color: stroke.color,
        canvasWidth: bufferCanvas.width,
        canvasHeight: bufferCanvas.height,
      };
      renderSegment(stroke.brush, rc, prev, curr);
    },

    resize() {
      syncSize();
    },

    getPlayheadPosition() {
      return playheadX;
    },

    setPlayheadPosition(x: number | null) {
      playheadX = x;
    },
  };
}

Right now it's just scattering circles around the point and drawing a fat connecting line — more like a stamp than a brush. A real round brush has a few key physical properties:

  • Persistent bristles — the same bristle positions frame-to-frame, not random each time
  • Bristles splay under pressure — low pressure = tight bundle, high pressure = spread fan
  • Each bristle drags independently — connecting from its previous position to its current one
  • Color loads and fades — opacity decreases over the stroke's life as "paint runs out"

The tricky part is that our renderer is stateless per-segment (it only sees prev and curr). To track persistent bristles we'd need per-stroke state. Two approaches:

  1. Fake it — use the stroke point index (derivable from timestamp) as a seed so bristle offsets are consistent between consecutive segments. Cheaper, no architecture change.
  2. Add brush state — pass a mutable state object per-stroke that tracks bristle positions. More realistic, but means changing the BrushRenderer signature.

Option 2 is cleaner and would pay off for all brush types. The change is small: BrushRenderer gets an optional state bag created on stroke start. Want me to go with that approach?
