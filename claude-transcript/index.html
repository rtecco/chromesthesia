Is there a way to make the "Round" tool more like a physical brush?

1 read

Right now it's just scattering circles around the point and drawing a fat connecting line — more like a stamp than a brush. A real round brush has a few key physical properties:

  • Persistent bristles — the same bristle positions frame-to-frame, not random each time
  • Bristles splay under pressure — low pressure = tight bundle, high pressure = spread fan
  • Each bristle drags independently — connecting from its previous position to its current one
  • Color loads and fades — opacity decreases over the stroke's life as "paint runs out"

The tricky part is that our renderer is stateless per-segment (it only sees prev and curr). To track persistent bristles we'd need per-stroke state. Two approaches:

  1. Fake it — use the stroke point index (derivable from timestamp) as a seed so bristle offsets are consistent between consecutive segments. Cheaper, no architecture change.
  2. Add brush state — pass a mutable state object per-stroke that tracks bristle positions. More realistic, but means changing the BrushRenderer signature.

Option 2 is cleaner and would pay off for all brush types. The change is small: BrushRenderer gets an optional state bag created on stroke start. Want me to go with that approach?

#222026-03-23T17:39:22.211Z

Explain the color / sound model a bit more to me.

The mapping works in three layers:

1. Color → Base Voice (HSL)

The color's HSL values determine the fundamental character of the sound:

  • Hue (what color it is) → waveform character. The hue is converted to a "warmth" score. Reds/oranges/yellows are warm → sawtooth waves (buzzy, brassy). Blues/purples are cool → triangle/sine waves (hollow, glassy). This also affects the filter type: warm hues get low-pass (muffled richness), cool hues get band-pass or high-pass (airy, open).

  • Saturation (how vivid) → harmonic richness. High saturation = more oscillator partials (up to 4), wider detuning between them (chorus-like fatness), more noise "breath". A pale lavender sounds pure and thin. A vivid cobalt blue sounds thicker and more complex.

  • Lightness (how bright/dark) → pitch. Dark colors are low-pitched (down to 55Hz), light colors are high-pitched (up to 880Hz). Titanium White is almost at the top of the range; a deep navy would rumble.

2. Position → Modulation (X/Y on canvas)

Where you paint shifts the voice in real-time:

  • X-axis → filter cutoff (brightness). Far left is very muffled/dark, far right is wide open/bright. This is an exponential curve so the middle of the canvas is fairly neutral.

  • Y-axis → pitch shift + reverb. Top of canvas shifts pitch up (up to +14 semitones) and is dry. Bottom shifts pitch down and drenches in reverb. So the top-right corner is the brightest, highest, driest point. Bottom-left is the darkest, lowest, most reverberant.

3. Brush → Effects Chain

The brush type adds a fixed effect on top of the voice:

  • Flat → reverb (cathedral wash)
  • Round → chorus (shimmery doubling)
  • Knife → distortion + slap delay (aggressive, edgy)
  • Dry → fast tremolo (gritty stutter)

In practice

So if you pick Cadmium Red (warm hue, high saturation, medium lightness) and paint with the Flat brush in the top-right corner: you get a mid-pitched buzzy sawtooth with lots of detuned partials, filter wide open, pitched up, barely any reverb, through a cathedral reverb effect.

Same color, bottom-left corner: same base voice but now muffled, pitched down over an octave, swimming in reverb.

Switch to Lavender (cool hue, low saturation, high lightness): now it's a high-pitched pure sine/triangle, thin, glassy — completely different character from the same position.

How hard is muting erased regions?

Moderate. The data is there — all strokes (paint and eraser) are stored in order with positions. The approach:

For each paint stroke point during replay, check if any later eraser stroke has a point close enough to overlap. If so, mute that voice/skip that point.

The tricky parts: - Proximity threshold — eraser widths vary with pressure (scraper: ~20-50px, solvent: ~25-55px). In normalized coords that's roughly 0.02-0.05. Fine for dozens of strokes, could lag with hundreds. - Granularity — since we play one voice per stroke, we can't mute the middle of a stroke easily. We'd either skip the whole stroke if mostly erased, or split it into segments around erased regions (more complex).

One cool feature would be: on replay, the stroke that's playing "shimmers" a bit. What's the complexity of that?

Low-moderate. The data is there — the replay system knows which stroke is active and has its points. The approach:

  1. Playhead exposes active stroke points — add a getter that returns the currently-playing stroke's point array (or null)
  2. Renderer draws a shimmer pass — after compositing the frame, trace the active stroke's path with an animated glow (e.g. sine-pulsing alpha, slightly wider, lighter color)
  3. Animation loop — the renderer already runs on rAF, so just add the shimmer draw call

The main nuance is that strokes are baked into the offscreen buffer — you can't modify them in place. But drawing on top with an additive/lighter blend is easy and looks good. Maybe 40-50 lines across playhead + renderer.
