Assistant2026-03-23T16:58:25.703Z

I see the problem — line 39: normalizePoint({ ...e, pressure: e.pressure } as PointerEvent). Spreading a PointerEvent doesn't work — DOM events aren't plain objects, their properties are on the prototype. So clientX, clientY, pressure are all undefined in the spread, and normalizePoint gets NaN coordinates.

Also, activeStroke is null when normalizePoint is called on line 39, so timestamp is always 0 for the first point — that's fine but the spread is the real bug.

Right now there are two separate audio modes:

Live audition (current) — Each brush stroke makes one continuous voice while you paint. It starts on pointer-down, modulates as you move, and stops on pointer-up. This is just tactile feedback — "hear what you're painting."

Playhead loop (Stage 4, not built yet) — When you press Play, a vertical line sweeps left-to-right across the canvas over 4 seconds. At each column it triggers voices for whatever strokes pass through that X position. The canvas becomes a sequencer: X = time, Y/color/brush = the sound. When it reaches the right edge, it loops back to the left.

So no, it shouldn't play the entire canvas on each stroke — just the stroke you're actively drawing. Does the current live audition feel right to you, or is there something off about it? If you're happy with it we can commit and move to Stage 4 to build the playhead loop.
